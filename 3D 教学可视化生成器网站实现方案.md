# 3D 教学可视化生成器网站实现方案

本项目旨在将 `AetherViz Master` 这一用于大模型的 3D 教育页面生成 skill 转化为一个可供教师线上使用的完整 Web 应用程序。用户只需输入想要讲解的关键词或点击预设的学科关键词，系统便会调用大语言模型（如 Gemini）并应用 `SKILL.md` 的指令生成一个零依赖、高性能 feather-light 的 3D 交互教学页面。

---

## 线上部署后端实现原理

因为 `SKILL.md` 主要是为 AI Agent 编写的角色说明，当我们将它部署到线上作为 Web 服务的后端时，实现原理如下：

1. **大模型 API 集成**：
   后端（Node.js/Express）将作为一个代理。它会读取本地的 `SKILL.md` 文件内容，将其作为 **System Instruction (系统提示词)** 传递给 LLM 接口。
2. **结构化 Prompt 组装**：
   当用户在前端输入关键词（如“光合作用”）时，后端将请求组装为：
   - **System Instruction**: `SKILL.md` 文件的完整文本（告知模型它是一个 3D 互动教育页面构建师，应如何使用 Three.js/SVG 渲染，输出规则等）。
   - **User Prompt**: `教学主题："${keyword}"。请严格按照规范生成该主题的单文件 HTML，绝不添加任何 markdown 代码块标记，不要包含 \`\`\`html 这样的包裹，也不要添加任何解释性文字，只输出 HTML 代码本身。`
3. **调用 Gemini 2.5 Flash**：
   使用 Google 的 **Gemini 2.5 Flash** 模型。该模型生成速度极快（通常在 10-15 秒内生成数千行复杂的 Three.js + HTML 代码），并且支持长上下文和极强的代码生成能力，非常契合 3D 生成的需求。
4. **代码清洗与流式输出**：
   即使我们在 Prompt 中强调“只输出 HTML 并且不要 markdown 块”，模型偶尔仍会返回被 \`\`\`html 括起来的代码。后端在接收到模型的响应后，将通过正则表达式自动剥离首尾的 markdown 代码块标记，确保返回给前端的是 100% 干净 of HTML 文本。
5. **部署方式**：
   - **后端环境**：Node.js 20+，使用官方的 `@google/genai` SDK。
   - **前端环境**：原生 HTML5 + CSS3 (Glassmorphism) + JavaScript。无需复杂的构建步骤，易于托管。
   - **线上托管**：可以部署到 Vercel、Zeabur、Railway、Docker 容器或任何支持 Node.js 的云服务器（如阿里云、腾讯云、AWS）。只需要在环境变量中配置 `GEMINI_API_KEY` 即可。

## 界面设计（Wow Factor & Premium Visuals）

为了让教师用户获得惊艳的视觉体验，我们将应用以下设计规范：
1. **深色赛博背景**：深海蓝与夜空灰的渐变背景 (`#0B0F19` 到 `#111827`)，配合半透明的科技感网格（Grid overlay）。
2. **玻璃拟态（Glassmorphism）面板**：控制台采用 `backdrop-filter: blur(12px)` 和微弱的白色半透明边框，搭配霓虹阴影。
3. **学科分类卡片**：
   - 物理（蓝色霓虹）、化学（橙红霓虹）、生物（翠绿霓虹）、数学（金黄霓虹）、天文（深紫蓝霓虹）、编程（青绿霓虹）。
   - 点击分类，下方会以平滑的滑入动画展示精选的 3D 生成关键词卡片（如：有丝分裂、黑洞、电磁感应等）。
4. **3D 原子旋转 Loading 动效**：当用户点击生成后，预览区域将被一个用 CSS 3D/Canvas 渲染的旋转原子（具有电子轨道和呼吸灯粒子效果）遮罩，显示“AetherViz AI 正在构建 3D 世界...”，让等待时间充满科技趣味。
5. **双栏式预览布局**：
   - **左栏（控制台）**：搜索输入框、学科分类、关键词列表、生成状态。
   - **右栏（实时预览区）**：一个精美的浏览器 Mockup 窗口，内置 `<iframe>`。生成完成后，HTML 会直接注入 iframe 中，教师无需下载即可在网页上直接操作、旋转 3D 模型、调节滑块、进行小测验。
   - **操作栏**：提供“新窗口全屏打开”、“复制 HTML 代码”和“下载 lesson.html 文件”按钮。

## Verification Plan

### 自动化与本地测试
1. **依赖安装验证**：
   运行 `npm install` 确保包正常解析。
2. **后端服务运行**：
   运行 `node server.js` 启动服务，验证静态页面是否能正常访问 (`http://localhost:3000`)。
3. **接口测试**：
   - 在前端点击“有丝分裂”或“牛顿第二定律”，检查网络请求 `POST /api/generate`。
   - 验证返回的响应是否是标准的 HTML 格式。

### 手动功能验证
- 验证在 iframe 中生成的 3D 模型是否可以使用鼠标进行旋转和缩放。
- 验证滑动条（如物理模拟中的参数）是否能实时更新 3D 画面。
- 验证点击“下载”按钮是否能将生成的页面保存到本地，并且本地双击打开后能独立完美运行。
- 验证在手机端打开时，触控旋转、双指缩放和响应式布局是否完好。
